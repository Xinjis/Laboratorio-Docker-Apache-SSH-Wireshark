Laboratorio de Captura de Tráfico

Autor: Alejandro Sánchez

Servicios expuestos: HTTP (80) SSH (2222)

Objetivo

Generar y capturar tráfico para analizar peticiones HTTP, autenticación básica y sesiones SSH con Wireshark.

Credenciales de prueba

Este sitio usa Basic Auth con usuario de laboratorio. En HTTP sin TLS la cabecera Authorization viaja en claro.

  • Usuario: admin
  • Contraseña: •••• (define con htpasswd)

Comandos útiles

Ejemplos para generar tráfico:

  • curl -v http://localhost --user admin:tu_clave
  • ssh demo@localhost -p 2222

Captura de tráfico

Para capturar dentro de los contenedores:

  • tcpdump -i eth0 -w /tmp/captura.pcap
  • Copiar al host: docker cp apache_server:/tmp/captura.pcap .

Abrir el archivo con Wireshark para analizarlo.

Qué buscar en Wireshark

  • HTTP: cabecera Authorization → credenciales en Base64.
  • SSH: handshake inicial y algoritmos de cifrado (el contenido está protegido).
  • TCP: handshakes 3-way y flags de cierre FIN/RST.
  • Comparativa: con TLS la cabecera Authorization ya no es visible.

Aviso

Solo con fines educativos. No usar credenciales reales. Habilita TLS para proteger credenciales en tránsito.

Nota: Este servicio usa autenticación básica (Basic Auth). El navegador enviará las credenciales en cada petición mientras dure la sesión y no existe un mecanismo de cierre de sesión desde la página.
